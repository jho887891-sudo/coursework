## 模块 2：协作策略对比

G4 方向 C 的核心实验——多 Agent 协作到底有没有用：

```
同一份 Benchmark (50 条)，四种策略：

┌─────────────────┬──────┬──────┬──────┬──────────┐
│ 策略              │ F1   │ Cost │ P99  │ 性价比    │
├─────────────────┼──────┼──────┼──────┼──────────┤
│ Single-Agent    │ 0.76 │ $0.12│ 2.1s │ 6.3 F1/$ │
│ Pipeline(串行)  │ 0.82 │ $0.31│ 6.5s │ 2.6 F1/$ │
│ Parallel-Vote   │ 0.85 │ $0.35│ 3.2s │ 2.4 F1/$ │
│ Debate          │ 0.88 │ $0.89│ 2.8s │ 1.0 F1/$ │
└─────────────────┴──────┴──────┴──────┴──────────┘

分析：
  - Debate F1 最高但成本是 Single 的 7.4 倍
  - Parallel-Vote F1=0.85, P99=3.2s, 性价比合理
  - Pipeline F1=0.82 比 Parallel-Vote 低 3%，但串行延迟是 2 倍
  → 推荐 Parallel-Vote 作为生产方案（F1 和延迟的最佳平衡点）
```

实验要求：
- 每种策略跑 30 次重复（同 Day 3 的 1500 次推理规模）
- 成本用 token 消耗量化（DashScope API 返回 usage.total_tokens）
- 延迟用 tracing span 记录每一步耗时

## 模块 3：竞品对标

```
同一份文档，三种审查者：

            人工(法律专业)   当前 Agent     GPT-4 直接审
            ─────────────    ─────────     ────────────
Precision        0.92           0.85           0.78
Recall           0.95           0.82           0.88
F1               0.93           0.83           0.83
幻觉率            0%             8%            15%
法规引用准确率    100%           91%            73%

分析：
  - 人工仍然是 Gold Standard——最高 P/R/F1，零幻觉
  - Agent 在 Precision 上优于 GPT-4(高7个点)——G5 的 Prompt 工程起作用了
  - GPT-4 在 Recall 上稍高(88% vs 82%)——通用能力强，但幻觉率也高
  - Agent 的法规引用准确率 91% vs GPT-4 的 73%——RAG 的检索能力在起作用
```

## 动手

### P0：评测闭环 Demo（小组必做）

1. **小型指标矩阵**：至少 2 个版本 × 3 个问题类型的 P/R/F1；
2. **不确定性**：对一个核心指标计算 Bootstrap CI；
3. **失败保留**：超时、解析失败等样本不得从分母中静默删除；
4. **简洁报告**：终端表格或 Markdown 即可。
3. **回归热力图**：二维 heatmap（Agent × Query Type）→ 标记哪些组合在最新版本退化
4. **幻觉率仪表盘**：按 Agent + 幻觉类型（法规/原文/数值）展示

### P1：协作策略对比（加分项）

四种策略的 F1 + 成本 + 延迟 三维对比 + Pareto 前沿分析

### P2：竞品对标（挑战项）

人工 vs Agent vs GPT-4 的三方对比（至少 5 份文档，每份至少跑 10 次）

| 验收项 | 权重 | 怎么测 |
|--------|------|--------|
| 三维 F1 矩阵 | 15% | 至少 27 个数据点 (3 Agent × 6 类型 × 3 行业?) |

| 回归热力图 | 10% | 二维 heatmap 可读 |
| 幻觉率报告 | 10% | 三种幻觉按 Agent 分列 |
| 协作策略对比 | 20% | 4 策略 × 3 维度 (F1/成本/延迟) |
| 竞品对标 | 15% | 三方 P/R/F1 + 差异分析 |
| 设计决策文档 | 20% | 指标选择/可视化设计/Pipeline 设计 |

## 设计决策文档

1. **为什么三维矩阵按 Agent × 类型 × 行业而不是按其他维度？** — 这三个维度对应不同的优化owner：G5 改 Prompt（Agent 维度）、G2 补规则（类型维度）、G3 补知识（行业维度）
2. **协作策略对比为什么推荐 Parallel-Vote 而不是 Debate？** — Debate F1 高 3% 但成本高 2.5 倍。除非客户愿意为 3% 的精度提升付 2.5 倍的钱
3. **Dashboard 为什么用终端 ASCII 而不是 Web？** — 终端版本 CI 可直接输出到 PR comment，零部署成本。Web 版作为进阶——项目后期 Demo 展示用

## 与标书审核项目的关系

这个实验用于理解 Agent 智能组评测体系的核心闭环，不等于完成正式 Benchmark 或生产 Dashboard。

你的 Dashboard → G4 的评测报告标准格式。你的协作策略对比 → G4 方向 C 的决策依据（选 Pipeline vs Parallel-Vote vs Debate）。你的竞品对标 → 项目对外展示的核心材料（"我们的 Agent 在法规引用准确率上超越 GPT-4 18 个百分点"）。

没有这套评测体系，你不知道 Agent 审得准不准、Prompt 改好了还是改坏了、协作有没有用、幻觉有多严重。有了它，G4 和 G5 的每一轮迭代都有数据说话。
