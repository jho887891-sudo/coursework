# 预备读本：开始写 Agent 前，你需要先知道什么

> 如果你第一次接触 LLM 和 Agent，请先完整读完本章，再进入第 1 课。这里不要求你已经会 Agent，只要求你会基本 Rust。

---

## 1. 先建立一张地图

这门课会反复出现六个词：

```text
模型 Model       根据输入生成候选输出
运行时 Runtime   控制模型何时调用、动作能否执行、何时停止
工具 Tool        Agent 可以请求执行的外部动作
环境 Environment 工具真正作用的对象，例如文件、数据库或用户
状态 State       当前任务已经知道的信息和剩余资源
评测 Eval        用一批任务判断系统是否可靠
```

最容易犯的错误，是把模型和 Agent 混为一谈。

```text
LLM：给它文本，它继续生成文本
Agent：把模型放进一个受约束的循环，让它观察、选择动作、接收结果并停止
```

因此模型回答错了，不一定只有“Prompt 不好”一个原因。还可能是：

- Runtime 把非法动作放行了；
- Tool 返回了错误数据；
- 检索没有找到相关证据；
- 状态丢失了关键约束；
- 预算太小，Agent 来不及完成任务；
- 评测数据本身标错了。

## 2. 一次 LLM 调用发生了什么

常见聊天接口接收一组消息：

```json
[
  {"role":"system","content":"你是一个课程助手"},
  {"role":"user","content":"解释 Rust 的 Result"}
]
```

模型根据这些输入生成下一段内容。

### 三种常见角色

| 角色 | 谁提供 | 用途 |
|---|---|---|
| system | 系统开发者 | 定义任务、边界和输出约束 |
| user | 用户 | 提出当前需求和提供数据 |
| assistant | 模型 | 模型之前生成的回复或动作建议 |

加入工具后还会出现 `tool` 消息，用来告诉模型工具执行结果。

### 模型不是数据库

模型生成的是“在当前上下文中看起来合理的下一段内容”，不是从数据库读取一条保证正确的记录。因此它可能：

- 记错事实；
- 编造不存在的来源；
- 输出格式接近 JSON 但无法解析；
- 对同一个问题给出不同答案；
- 声称完成了它实际没有执行的动作。

这也是课程要学习证据、校验、Trace 和 Eval 的原因。

### 上下文不是永久记忆

模型只能看到本次请求发送给它的消息。程序关闭后，如果你没有保存状态，下次模型不会自动知道之前发生了什么。

即使发送了完整历史，也有三个问题：

- 上下文长度有限；
- 历史越长，成本和延迟越高；
- 旧信息可能干扰当前任务。

## 3. 结构化输出

Agent Runtime 通常不希望模型只返回散文，而希望它提出一个可解析动作：

```json
{
  "action": "search_rules",
  "arguments": {
    "query": "投标保证金比例"
  }
}
```

但你必须假设模型有时会返回：

```text
我建议先搜索一下相关规定。
```

或者：

```json
{"action":"search_rules","arguments":{}}
```

第二段虽然是合法 JSON，但缺少 `query`，仍然不是合法动作。

所以处理流程应当是：

```text
模型文本
  → JSON 解析
  → 字段和类型校验
  → 动作是否注册
  → 权限和预算检查
  → 执行
```

## 4. Tool Calling 的完整闭环

模型不会真的执行 Rust 函数。它只能表达“我想调用某个工具”。

```text
第 1 次模型调用
用户：读取 rules.txt 并总结
模型：调用 read_file(path="rules.txt")

你的程序
解析动作 → 检查路径 → 执行 read_file → 得到文本

第 2 次模型调用
程序把工具结果发给模型
模型：基于文件内容生成总结
```

关键边界：

- 模型决定“建议调用什么”；
- Runtime 决定“允不允许调用”；
- Tool 决定“如何执行”；
- Environment 返回“真实发生了什么”。

## 5. 为什么需要 FakeModel

如果每个测试都调用真实 API，会出现：

- 结果不稳定；
- 测试慢；
- 消耗费用；
- 网络失败影响测试；
- 很难稳定复现非法 JSON、重复动作等边界情况。

因此我们为模型定义接口：

```rust
#[async_trait::async_trait]
trait Model {
    async fn generate(&self, input: &ModelInput) -> anyhow::Result<String>;
}
```

真实实现访问 LLM，测试实现按顺序返回预设文本：

```rust
let fake = FakeModel::new(vec![
    r#"{"action":"echo","arguments":{"text":"hello"}}"#,
    r#"{"action":"finish","arguments":{"answer":"done"}}"#,
]);
```

这不是“模拟模型智能”，而是在测试你自己的 Runtime 遇到某种模型行为时是否正确。

## 6. 本课需要的 Rust 知识

### enum：描述有限状态和动作

```rust
enum RunStatus {
    Running,
    Completed,
    Failed(String),
}
```

### Result：区分成功与失败

```rust
fn parse_action(text: &str) -> Result<Action, ParseError> {
    // 解析失败时返回 Err，不 panic
}
```

### trait：替换模型和工具实现

```rust
trait Tool {
    fn name(&self) -> &'static str;
    fn execute(&self, args: Value) -> Result<Value, ToolError>;
}
```

### async：等待模型和工具

```rust
let response = model.generate(&input).await?;
```

### serde：结构化数据

```rust
#[derive(Serialize, Deserialize)]
struct ProposedAction {
    action: String,
    arguments: serde_json::Value,
}
```

如果这些代码完全陌生，先复习 Rust 的 enum、trait、Result、测试和 async，再继续课程。

## 7. 你会反复写的项目目录

推荐每个 Lesson 使用相似结构：

```text
src/
├── model.rs        Model + FakeModel
├── runtime.rs      Agent 主循环和预算
├── state.rs        状态与事件
├── action.rs       类型化动作
├── tools.rs        Tool trait 与 Registry
├── trace.rs        结构化运行轨迹
└── main.rs         命令行入口
tests/
├── runtime_tests.rs
├── tool_contract_tests.rs
└── trajectory_tests.rs
```

不要第一天就把所有东西写进 `main.rs`。

## 8. 如何阅读每一课

每课建议按以下顺序：

1. 先读“问题场景”，理解为什么需要该机制；
2. 用自己的话解释核心术语；
3. 手工推演 worked example；
4. 跟着“课堂练习”实现最小版本；
5. 通过检查点后再进入作业；
6. 先写实验预测，再运行测试；
7. 最后完成自测题和报告。

如果你直接跳到作业，很容易写出能运行、但解释不了为什么的代码。

## 9. 入门自测

在进入第 1 课前，确认你能回答：

1. LLM 和 Agent 是同一个概念吗？
2. 模型返回合法 JSON，动作就一定合法吗？
3. 模型为什么不能直接读取你的文件？
4. System Prompt 能否替代操作系统权限？
5. FakeModel 测试的是什么？
6. 工具执行失败应该 panic 还是返回 Observation？

参考答案：依次为“不一定”“不一定”“需要宿主程序提供工具”“不能”“Runtime 对模型行为的处理”“返回结构化错误供 Runtime 决策”。

## 10. 推荐学习资料

- Rust Book：枚举、错误处理、trait、测试；
- Tokio Tutorial：异步任务、超时与 channel；
- Serde / serde_json 文档：结构化输入输出；
- ReAct 论文：理解 Reason + Act 的交替思想；
- OWASP LLM 应用安全资料：理解 Prompt Injection 和过度授权。

这些是延伸材料。每课的核心任务不要求你先读完整论文才能开始。
