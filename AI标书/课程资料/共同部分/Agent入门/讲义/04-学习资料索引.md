# 学习资料索引：每课该读什么、读到什么程度

> 资料按“必须 / 选读”分层。必须资料用于补足课程基础；选读用于想继续深入的同学。不要为了读完所有论文而推迟动手实验。

---

## 课程开始前

### 必须

1. [The Rust Programming Language](https://doc.rust-lang.org/book/)
   - 重点：enum、trait、Result、测试；
   - 读完后应能用 `Result<T, E>` 表达可恢复错误。
2. [Rust Book：Error Handling](https://doc.rust-lang.org/book/ch09-00-error-handling.html)
   - 思考：Tool timeout 属于可恢复错误还是程序 bug？
3. [Tokio Tutorial](https://tokio.rs/tokio/tutorial)
   - 重点：async/await、异步任务和 channel 的基本语义；
   - 不要求入门课开始前掌握复杂并发。
4. [Serde](https://serde.rs/)
   - 重点：`Serialize`、`Deserialize` 和错误处理。

### 检查

阅读后尝试独立完成：

```rust
#[derive(Serialize, Deserialize)]
struct Action {
    name: String,
    arguments: serde_json::Value,
}
```

并为非法 JSON 写一个测试。

### 阅读问题

1. 为什么 Prompt 不能成为权限边界？

### 选读

### 阅读问题

## 如何读去论文?

入门阶段不用逐公式阅读。采用四遍法：

1. 标题、摘要：论文解决什么问题？
2. 架构图：输入、状态、动作和输出是什么？
3. 实验表：和什么 baseline 比？指标是什么？
4. 限制：在哪些任务、模型和数据上验证？

最终写下：

```text
Claim：论文声称什么？
Evidence：哪张表或哪个实验支持？
Boundary：这个结论不能自动推广到哪里？
```
