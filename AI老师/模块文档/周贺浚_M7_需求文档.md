# M7 即时交互式问答 —— 需求文档


---

## 一、定位

M7 是**横向服务层**，为 M1(做题)/M2(讲题)/M4(讲课)/M5(备课) 提供统一的实时问答能力。**核心职责：低延迟、高可用、可打断、可恢复。**

---

## 二、工作范围


### ~~2.？ 前端（React）（可能不是我们负责而是前端组负责）~~

### 2.1 后端（Java Spring Boot）

| # | 模块 | 核心内容 |
|---|------|---------|
| 1 | **SSE流式基础设施** | Spring WebFlux `Flux<ServerSentEvent>` 透传Python流 |
| 2 | **统一对话API** | `POST /api/conversation/ask`（SSE流式） |
| 3 | **打断机制** | `POST /api/conversation/interrupt` → 取消Flux + 通知Python |
| 4 | **会话管理** | Redis存活跃会话（`user:session:{sessionId}`，TTL 30min），MySQL全量持久化 |
| 5 | **限流** | Redis滑动窗口 `ai:ratelimit:{userId}`，10次/分钟 → HTTP 429 |
| 6 | **重试+熔断+降级** | 失败重试3次（1s/3s/9s），超时30s熔断，兜底模板回答 |
| 7 | **监控日志** | `ai_call_logs` 表记录每次调用（延迟、token、成功率） |
| 8 | **缓存** | Redis `qa:cache:{questionHash}` 高频问题缓存，TTL 1h |
| 9 | **请求合并** | 同一会话200ms内相同问题去重 |
| 10 | **会话CRUD** | 创建/查询/删除会话 |
| 11 | **监控统计API** | 总体/按模块的性能概览接口 |
| 12 | **WebSocket**（P2） | 双向通信通道，心跳保活，断线重连 |

### 2.2 依赖的Python AI服务

| 依赖 | 接口 | 用途 |
|------|------|------|
| AI流式生成 | `POST /api/internal/ai/generate/stream` | 后端调用此接口获取AI回复流 |
| 上下文压缩 | `POST /api/internal/ai/compress-context` | 历史消息超限时压缩摘要 |

## 三、数据表设计

### 3.1 conversation_sessions（已有）
```sql
id, user_id, scene_type, context_json, status(active/paused/ended), created_at, updated_at
```

### 3.2 conversation_messages（已有）
```sql
id, session_id, role(user/assistant/system), content, message_type, metadata_json, created_at
```

### 3.3 ai_call_logs（新增）
```sql
id, call_id, module_name, scene_type, user_id, session_id,
prompt_length, response_length, latency_ms, ttfb_ms,
token_input, token_output, success, error_msg, interrupted, created_at
```

### 3.4 Redis
| Key | TTL | 用途 |
|-----|-----|------|
| `user:session:{sessionId}` | 30min | 会话状态 |
| `ai:ratelimit:{userId}` | 1min | 限流 |
| `qa:cache:{questionHash}` | 1h | 答案缓存 |
| `user:profile:{userId}` | 30min | 画像缓存（消费M6） |

## 五、验收标准

| 验收项 | 标准 | 优先级 |
|--------|------|--------|
| SSE流式响应 | TTFB < 2s，打字机效果 | P0 |
| 打断 | 500ms内停止生成，保留已生成内容 | P0 |
| 多轮上下文 | AI记住前文，正确指代消解 | P0 |
| 断线恢复 | 刷新后历史消息完整恢复 | P0 |
| 降级 | AI异常时返回兜底回答，不白屏不报错 | P0 |
| 限流 | 超10次/分钟 → HTTP 429提示 | P0 |
| 重试 | 失败自动重试最多3次 | P1 |
| 缓存命中 | 高频问题走Redis，不重复调AI | P1 |
| 会话超时 | 30min无交互自动结束 | P1 |
| 性能可监控 | 可查看各模块AI调用延迟/成功率/token | P1 |
| 请求合并 | 200ms内相同问题去重 | P1 |
| WebSocket | 连接+心跳+重连正常（P2） | P2 |

### 性能目标
| 指标 | 目标 | 告警 |
|------|------|------|
| TTFB | < 2s | > 3s |
| P50响应 | < 3s | > 5s |
| P95响应 | < 8s | > 10s |
| 成功率 | > 97% | < 95% |
| 打断响应 | < 500ms | > 1s |
