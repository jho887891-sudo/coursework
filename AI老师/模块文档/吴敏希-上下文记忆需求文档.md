## 1. 文档概述
### 1.1 文档目的
本文档为 LeapMind 平台 M6 上下文记忆模块完整需求说明，定义模块能力、数据链路、交互逻辑、技术指标与验收标准，作为开发、测试、算法落地唯一依据。

### 1.2 模块核心价值
搭建平台统一底层记忆中枢，解决 AI 无长效记忆、教学内容千人一面问题；采集全平台学生学习行为构建多维学情画像，在所有 AI 交互场景自动注入学生历史数据，实现个性化教学；依托遗忘曲线算法自动推送复习任务，打造平台 “越用越聪明” 核心差异化壁垒。

### 1.3 版本说明
V1.0：首期完整落地版本，覆盖全链路记忆采集、分层存储、画像计算、记忆注入、遗忘复习调度、标准化对外 API。

## 2. 项目背景与模块定位
### 2.1 现有产品痛点
1. AI 交互无长效记忆，每次对话、讲课、答疑重置上下文，无法识别学生历史薄弱知识点、错题、学习偏好；
2. 所有学生接收统一标准化讲解，几何薄弱、代数薄弱学生无差异化教学方案；
3. 各业务模块数据割裂，做题、讲题、课堂交互数据无法互通，缺少统一学情诊断依据；
4. 无知识点遗忘管理机制，学生反复遗忘同类知识点，系统不会主动推送针对性复习；
5. 缺少完整用户学习画像体系，竞品暂无全链路长效记忆能力，是平台核心竞争壁垒。

### 2.2 M6模块定位
M6 是平台底层记忆系统，承担四大核心职责：
1. 统一记忆采集：汇总 M1/M2/M4/M7/M8 全模块学生学习行为数据；
2. 多维学情画像计算：结构化沉淀知识点掌握程度、错题、学习偏好、答题节奏；
3. 上下文记忆分发注入：所有 AI 讲课、答疑、对话交互自动读取画像，生成个性化内容；
4. 遗忘周期智能调度：基于艾宾浩斯遗忘曲线计算复习节点，联动做题模块推送复习任务。

### 2.3 模块交付最终效果
AI 拥有持续长效记忆，每一次学生交互都会优化画像精度；跨会话、跨日期交互可自动识别学生薄弱项与学习习惯；系统自动生成精准薄弱点，主动推送复习计划，实现从通用教学到千人千面个性化教学。

## 3. 关联模块与依赖关系
### 3.1 上游输入模块（数据写入M6）
M1 做题板块、M2 拍照答疑 + AI 讲题、M4 即时讲课、M7 实时打断提问、M8 虚拟 AI 教师形象
上报数据包含：答题记录、错题、听懂 / 不懂反馈、课堂互动、语音 / 文字交互、教师形象音色偏好等。

### 3.2 下游调用模块（读取M6记忆数据）
M1、M2、M3 薄弱点列表、M4、M7、M8
所有 AI 生成内容前，自动拉取 M6 学生画像作为上下文输入。

### 3.3 强依赖约束
M6 画像生成依赖 M1 做题模块答题数据，无答题记录则知识点掌握度无法量化，个性化教学效果大幅下降。

## 4. 目标用户与核心业务场景
### 4.1 核心用户
K12 阶段在校学生（小学、初中、高中）

### 4.2 典型业务场景
1. **场景 1：跨天连贯答疑，自动复用历史薄弱信息**
学生首日提问勾股定理，系统记录其几何薄弱，采用图形拆解讲解；3 天后再次提问同一知识点，AI 自动读取 M6 记忆，无需学生重复说明短板，直接沿用图示化讲解逻辑。

2. **场景 2：错题反馈反向更新记忆，同步全模块个性化策略**
学生使用 M2 拍照讲题后选择 “没听懂”，M6 自动标记该知识点为高频困惑项；同步至 M3 生成薄弱点、M4 后续讲课自动增加例题、放慢讲解节奏。

3. **场景 3：AI讲课动态适配学生学情**
上传课件启动 M4 即时讲课，M6 自动注入学生知识图谱，课件内对应薄弱难点自动增加拆解步骤、插入互动提问。

4. **场景 4：遗忘曲线自动推送专项复习任务**
学生 3 天前做错二次函数题型，系统判定到达最佳复习时间，M6 推送复习信号至 M1 做题板块，首页生成专项薄弱练习题。

5. **场景 5：自适应匹配学生学习交互偏好**
学生长期使用语音提问、偏好图文讲解；M7对话、M2讲题、M4 讲课会自动增加图示内容，优先提供语音讲解，减少纯文字输出。

## 5. 专业术语定义
1. **短期会话记忆**：单次对话窗口临时上下文，存储最近 20 轮交互，支撑实时连续追问；页面刷新、断线可完整恢复，会话结束后仅高价值教学数据归档至长期记忆。
2. **长期学情记忆**：持久化存储学生全维度学习数据，分为四大数据库：知识点掌握图谱库、错题困惑库、学习偏好库、遗忘周期调度库。
3. **记忆过滤器**：自动过滤闲聊、无效重复提问，仅留存知识点、错题、学习反馈等高价值教学数据。
4. **多维用户画像**：M6 聚合全量学习数据生成，包含知识点掌握分值、Top5 薄弱点、学习偏好、高频困惑知识点、待复习清单。
5. **知识点掌握度**：分值区间 0~100，0-40 红色薄弱、41-70 黄色一般、71-100 绿色熟练，同步 M3 薄弱点可视化图谱。
6. **记忆注入**：AI 发起讲课、答疑、对话前，拉取学生画像作为上下文传入大模型，实现差异化输出。
7. **遗忘曲线调度器**：内置间隔重复算法，根据知识点掌握等级自动计算最优复习间隔。

## 6. 整体架构设计
M6内部三层分层架构：
1. **记忆采集层**
统一接收各模块上报数据，记忆过滤器清洗无效信息，结构化处理后写入存储库；提供标准化上报 API，统一数据格式。

2. **画像计算层**
实时计算知识点掌握分值、薄弱点排序、学习偏好标签、知识点复习间隔；定时任务批量更新全量学生复习计划。

3. **记忆调度输出层**
管理短期会话上下文；对外提供画像读取、复习查询通用 API；将学生画像数据下发至各业务模块完成记忆注入。

## 7. 详细功能需求（核心功能）
### 功能 1：全模块统一记忆采集与数据过滤
- 多源数据标准化接入，支持 M1/M2/M4/M7/M8 所有学习行为上报；
- 过滤规则：保留知识点提问、错题、听懂 / 不懂反馈、练习记录、交互偏好；丢弃无意义闲聊、空白输入、重复无效提问；
- 自动抽取题目绑定知识点、错误类型、学生困惑关键词；
- 每条记忆记录携带时间戳、来源模块、场景标签；
- 验收标准：上游模块操作完成后，1 秒内完成数据入库。

### 功能 2：分层记忆存储体系（短期会话记忆+长期四大记忆库）
#### ①短期会话记忆
- 存储单会话最近 20 轮学生与 AI 交互记录；
- 会话内实时读取，支撑 M7 实时打断、连续追问；
- 页面刷新、网络断线后完整还原对话历史；
- 会话关闭后筛选高价值教学数据归档，冗余对话自动清理。

#### ②长期学情四大存储库
A. **知识点掌握图谱库**
绑定 K12 全学段知识点与学生掌握分值，每次答题、讲题反馈自动更新分值，输出三色掌握等级标签。

B. **错题与困惑记录库**
存储历史错题、错误原因、标记 “听不懂” 的知识点，统计同一知识点出错频次，标记高频薄弱项，关联 M2 讲解记录支持回溯。

C. **学习偏好标签库**
自动沉淀三类标签：内容偏好（图示 / 文字 / 语音）、交互偏好（语音输入 / 文字 / 拍照）、节奏偏好（答题快慢、打断频率）、虚拟教师形象音色偏好。

D. **遗忘周期调度库**
记录每个知识点掌握等级、上次复习时间、系统计算的下次复习时间。

### 功能 3：多维学生画像自动计算引擎
A. **知识点掌握度计分规则**
答对题目 + 5~10 分，答错 - 10~15 分；反馈 “听不懂” 额外扣 10 分；连续答对同类知识点稳定加分。

B. **薄弱点排序逻辑**
综合错误率、提问频次、听不懂反馈，自动生成 Top5 薄弱知识点，同步 M3 模块。

C. **学习偏好动态更新**
以近 7 天交互行为为高权重，自动更新偏好标签，实时同步至画像接口。
标准化统一画像输出结构，见附录 JSON 示例。

### 功能 4：全场景自动记忆注入，个性化差异化输出
A. 触发场景：M2 拍照讲题、M4 即时讲课、M7 对话答疑、M8 虚拟教师交互；
B. 执行逻辑：业务模块调用画像接口获取学情，将画像作为上下文传入大模型；
C. 差异化输出规则：
- 几何薄弱：优先图形、图示拆解，减少纯公式推导；
- 代数熟练：简化图形，侧重公式推导；
- 高频困惑知识点：增加分步拆解、易错提醒、同类例题；
- 偏好语音 / 图文：输出内容增加可视化描述，引导语音讲解；
D. 验收标准：同一问题，不同学情学生获得差异化讲解内容，无统一模板输出。

### 功能 5：艾宾浩斯遗忘曲线复习调度
A. **复习间隔规则**
- 薄弱（0-40 分）：1 天、3 天、7 天；
- 一般（41-70 分）：3 天、7 天、15 天；
- 熟练（71-100 分）：15 天、30 天，降低复习优先级；
B. 定时任务每日扫描全学生待复习知识点；
C. 推送复习数据至 M1 做题板块，自动生成专项练习题；
D. 学生完成复习后，M1 上报答题结果，M6 重新更新掌握分值与下一轮复习时间；
E. 逾期未复习自动提升复习优先级，延后复习周期。

### 功能 6：标准化对外 API 接口
1. 画像读取接口：传入学生 ID，返回完整多维学情画像；
2. 记忆上报接口：各模块上报学习行为，写入 M6 存储；
3. 复习任务查询接口：查询学生当日待复习知识点清单。

## 8. 业务完整流程
### 流程 1：学习行为采集->画像更新
1. 学生在 M1/M2/M4/M7 完成答题、讲题、课堂交互；
2. 业务模块调用记忆上报接口推送行为数据；
3. M6记忆过滤器清洗有效教学数据；
4. 结构化数据写入四大长期记忆库；
5. 画像引擎重新计算知识点掌握度、薄弱点、偏好标签；
6. 更新并缓存学生完整多维画像，供下游模块调用。

### 流程 2：AI交互自动注入记忆上下文
1. 用户发起讲题、讲课、对话提问；
2. 业务模块调用 M6 画像读取接口；
3. M6 返回当前学生完整学情画像；
4. 业务模块将画像传入大模型上下文；
5. AI 基于学生薄弱、偏好生成个性化讲解；
6. 交互结束后，听懂 / 不懂、答题结果回传 M6，更新记忆数据。

### 流程 3：遗忘曲线复习推送全流程
1. 系统每日定时任务批量扫描全学生知识点数据；
2. 匹配每个知识点预设复习时间节点；
3. 筛选当日到达复习周期的知识点；
4. 将复习清单推送至 M1 做题板块；
5. 学生完成专项复习答题，M1 上报答题数据；
6. M6 更新知识点掌握分值，重新计算下一次复习间隔。

## 9. 非功能需求（性能、稳定、安全、可观测）
### 9.1 性能需求
1. 画像读取接口响应延迟≤200ms；
2. 记忆数据上报写入延迟≤1s；
3. 支持万级学生并发读写，无服务阻塞；
4. 页面刷新后短期会话记忆恢复≤300ms。

### 9.2 稳定性需求
1. 学生记忆数据持久化存储，数据库双备份，杜绝学情数据丢失；
2. M6 服务异常触发降级策略：关闭个性化记忆，返回通用教学内容，不阻断 M1/M2/M4 核心业务流程；
3. 网络断线、页面刷新后短期对话记忆完整恢复。

### 9.3 安全与隐私需求
1. 学生记忆数据按学生 ID 隔离，禁止跨用户读取画像；
2. 支持学生一键清空全部记忆画像、重置所有学情数据；
3. 全量学生学情数据加密存储，符合青少年个人信息保护规范；
4. 后台仅可查看汇总统计数据，禁止导出单学生完整记忆详情。

### 9.4 可观测需求
1. 监控所有 API 调用量、响应延迟、失败率；
2. 完整记录画像更新日志、遗忘复习调度执行日志；
3. 配置异常告警：接口超时、数据写入失败、批量任务中断。

## 10. 验收标准与交付成果
### 10.1 功能验收指标
1. AI 具备跨会话、跨日期长效记忆，可自动识别学生历史薄弱点与学习偏好；
2. 系统输出 Top5 薄弱知识点与学生真实错题分布匹配度≥90%；
3. 相同问题针对不同学情学生可输出差异化讲解内容，人工测试可直观感知个性化；
4. 遗忘曲线调度正常运行，到期自动推送薄弱知识点专项复习任务；
5. M1/M2/M3/M4/M7/M8 全模块可正常调用 M6 记忆数据，数据链路完整打通。

### 10.2 性能验收指标
接口延迟、并发承载、断线恢复全部满足性能指标；服务异常降级策略生效，无页面白屏、无限等待问题。

### 10.3 模块最终 Result
AI 拥有持续记忆能力，每一次交互持续优化画像精度，平台实现 “越用越聪明”；构建竞品短期难以复制的底层记忆壁垒，支撑全平台个性化教学核心能力。

## 11. 风险、约束与迭代规划
### 11.1 业务约束
1. M6 依赖 M1 做题模块提供基础答题数据，无答题数据则画像量化失效；
2. V1.0 首期仅支持数学、物理、英语 K12 主流学科，其余学科后续迭代扩展；
3. 记忆过滤规则仅保留教学相关数据，闲聊内容不入库。

### 11.2 潜在风险
1. **学生交互数据量过大导致存储压力上涨**
应对：增加数据冷热分层存储，过期低价值对话自动归档压缩。
2. **高并发场景画像计算延迟升高**
应对：引入画像缓存机制，定时批量更新，减少实时计算压力。

### 11.3 迭代规划
1. V1.0：完成基础记忆采集、分层存储、画像计算、记忆注入、遗忘复习调度；
2. V1.1 迭代：新增学生记忆可视化面板、家长端学情记忆查看、多学科全量适配。
